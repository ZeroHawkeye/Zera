This document is automatically generated from the OpenAPI specification and contains detailed descriptions of all backend API endpoints.

Zera backend provides API services using [Connect-RPC](https://connectrpc.com/), with all interfaces supporting gRPC, gRPC-Web, and Connect protocols.

- **AuthService** - User login, registration, logout, and token refresh
- **GreetService** - Example service demonstrating basic usage

All requests must include the following headers:

| Header | Description | Required |
|--------|-------------|----------|
| `Connect-Protocol-Version` | Connect protocol version, fixed value `1` | Yes |
| `Connect-Timeout-Ms` | Request timeout in milliseconds | No |
| `Authorization` | Authentication token (format: `Bearer <token>`) | Required for protected endpoints |

All interfaces use the standard Connect error format:

```json
{
  "code": "not_found",
  "message": "资源不存在"
}
```

Error codes include: `canceled`, `unknown`, `invalid_argument`, `deadline_exceeded`, `not_found`, `already_exists`, `permission_denied`, `resource_exhausted`, `failed_precondition`, `aborted`, `out_of_range`, `unimplemented`, `internal`, `unavailable`, `data_loss`, `unauthenticated`
