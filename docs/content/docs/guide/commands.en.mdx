Zera uses [Taskfile](https://taskfile.dev) as its task runner, offering a rich set of development commands.

## Development Mode

| Command | Description |
|---------|-------------|
| `task dev` | Start the full development environment, auto-generate proto code and launch both frontend and backend services (mprocs TUI) |
| `task dev:frontend` | Start only the frontend development service |
| `task dev:backend` | Start only the backend service |

### task dev

This is the most commonly used development command. It will:

1. Automatically run proto code generation
2. Use [mprocs](https://github.com/pvolok/mprocs) to start both frontend and backend services simultaneously
3. Provide a unified TUI interface to manage all processes

```bash
task dev
```

## Proto Workflow

| Command | Description |
|---------|-------------|
| `task proto` | Complete proto workflow: lint → format → generate → check |
| `task proto:lint` | Check proto file compliance |
| `task proto:format` | Format proto files |
| `task proto:breaking` | Check for breaking changes in proto (compared against main branch) |

### task proto

Run the complete proto workflow to ensure code quality:

```bash
task proto
```

This command executes the following steps in sequence:
1. **lint**: Check whether proto files comply with specifications
2. **format**: Automatically format proto files
3. **generate**: Generate code for both frontend and backend
4. **check**: Verify that the generated code compiles correctly

## Code Generation

| Command | Description |
|---------|-------------|
| `task proto:gen` | Generate all proto code (frontend and backend in parallel) |
| `task proto:gen:backend` | Generate only backend Go code |
| `task proto:gen:frontend` | Generate only frontend TypeScript code |

### Generate Separately

When you only need to update code for one side:

```bash
# Generate backend code only
task proto:gen:backend

# Generate frontend code only
task proto:gen:frontend
```

## Code Validation

| Command | Description |
|---------|-------------|
| `task proto:check` | Validate correctness of generated code (frontend and backend in parallel) |
| `task proto:check:backend` | Check backend Go code compilation |
| `task proto:check:frontend` | Check frontend TypeScript types |

## Watch Mode

| Command | Description |
|---------|-------------|
| `task proto:watch` | Watch for proto changes and auto-generate + validate |
| `task proto:watch:fast` | Watch for proto changes and generate only (skip validation, faster) |

### Using Watch Mode During Development

During development, you can use watch mode to automatically regenerate code:

```bash
# Full watch (includes type checking)
task proto:watch

# Fast watch (skips type checking, faster)
task proto:watch:fast
```

<Callout type="info">
  We recommend using `task proto:watch:fast` during development for faster feedback loops.
</Callout>

## Common Development Workflows

### 1. Daily Development

```bash
# Start development environment
task dev
```

### 2. Modifying APIs

```bash
# 1. Modify proto files
# 2. Run complete proto workflow
task proto

# 3. Implement backend handlers
# 4. Update frontend calls
```

### 3. Adding New Services

```bash
# 1. Create new proto file
# 2. Generate code
task proto:gen

# 3. Implement business logic
# 4. Register service
```
